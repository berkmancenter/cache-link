Abstract

The mset attribute intended to give authors a way to provide temporal context to the target content of a hyperlink, links to copies of the target content of a hyperlink, or both.

The primary purpose of the mset attribute is to address [Internet censorship] and [link rot].

1 Introduction

This specification provides developers with a means to declare:

  • the date at which target content of a hyperlink was referenced in the context of the time the current document, article or section was authored
  • the location of copies of the target content of a hyperlink and, optionally, the date the copy was made
  • both the temporal reference and location of a copy
  • optionally, if the location is specified, the relationship of the copy to the target content

This is achieved by introducing mset to HTML, as an attribute for the a element to support specifying dates and multiple copies of an external resource. Copies can vary in presentation, format, and origin.

The a element ramains backwards compatible with legacy user agents which will ignore the mset attribute.

The mset attribute differs significantly from the link element (which also relates documents to each other) in that the mset attribute relates the target of the a element to other documents where the link element relates the current document to other documents.

1.1 When to use mset

The mset attribute can be used any time a publisher authors a document which references other content via hyperlinks. At its most basic, the publisher can provide only the date when target content was referenced while authoring a document.

If one ore more copies of the target content are known to exist, the mset attribute should be used embed the locations of these copies into the a element in the current document.

The mset attribute is not a replacement for the href attribute. When the original date of reference is unknown and there are no known copies of target content, developers should omit the mset attribute.

1.2 Examples of usage

The mset attribute can be set to one or more [cache candidate string]s separated by "," (U+002C) characters. These examples show some (but not all) variations of the mset attribute.

This example shows the most basic usage of the mset attribute to declare the date external content was referenced while authoring the current document. The locations of existing copies can be omitted.

4 Additions to the a Element

The a element represents a hyperlink to other parts of the current document or a hyperlink to other documents.

A hyperlink is determined by the presence of a [URL] in the href attribute.

4.1 The mset Attribute

Name: mset

The mset attribute...

  • either date of reference or location of copy
